0 / 5
1 ✓
Clonar el repositorio
git clone · cd GameLink-Neo4J
▾
bash 
git clone https://github.com/slayaglez/GameLink-Neo4J.git
cd GameLink-Neo4J
info Clona el repositorio con toda la configuración de Docker y los datos de Cypher incluidos.
2 ✓
Crear el archivo .env
cp .env.example → .env
▾
bash 
cp .env.example .env
importante Si necesitas cambiar credenciales u otras variables, edita el .env antes de continuar.
3 ✓
Crear carpetas de Neo4j
data · logs · plugins
▾
bash 
mkdir -p data logs plugins
info Docker necesita estos directorios para montar volúmenes. Sin ellos, Neo4j no puede persistir datos ni escribir logs.
4 ✓
Levantar Neo4j con Docker
docker compose up -d
▾
bash 
docker compose up -d
info El flag -d (detached) levanta el contenedor en segundo plano. Puedes verificar el estado con docker ps o docker compose ps.
5 ✓
Cargar los datos en Neo4j
esperar healthy · cypher-shell · datos.cypher
▾
⚠ espera Antes de ejecutar, espera a que el contenedor esté healthy. Puedes comprobarlo con docker ps — la columna STATUS debe mostrar (healthy).
bash 
docker exec -it neo4j-gamelink cypher-shell \
  -u neo4j -p gamelink1234 \
  --file /var/lib/neo4j/import/datos.cypher
resultado esperado Deberían aparecer muchos mensajes de tipo Added X nodes / relationships. Si ves los INSERTs fluyendo, ¡todo está funcionando!
